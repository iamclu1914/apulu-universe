# Vawn Research

Entry point for all formal Vawn research work: daily pipeline output, APU-xxx tickets, competitive scans, partnership decks, streaming strategy, sync opportunities.

For the *synthesized* knowledge layer, go to [[../../wiki/vawn-project/_index]]. Research here is raw — wiki is digested.

## When to reference this hub

- You want today's discovery, ideation, or briefing output
- You're working a research ticket (APU-xxx)
- You're looking for competitive, partnership, streaming, sync, or prompt-research context specific to Vawn
- You want to know which output lives where after the 2026-04-14 migration out of `journals/vawn/`

## Naming convention (and the drift)

Two naming schemes coexist in this tree:

- **Kebab-case with date prefix:** `2026-04-17-competitive-report.md`, `2026-04-17-playlist-monitor.md`
- **Title case with em-dash date suffix:** `Daily Briefing -- 2026-04-20.md`, `X Pipeline -- 2026-04-20.md`

Additionally, files from on/before 2026-04-09 use the real em-dash character (`—`) while later files use double-hyphens (`--`). This breaks plain-text search. Going forward, prefer **`YYYY-MM-DD-kebab-slug.md`** for all new dated notes.
