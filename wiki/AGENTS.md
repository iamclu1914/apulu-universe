# wiki/ — Codex Instructions

`wiki/` is the LLM-compiled knowledge base — digested, cross-linked, hub-organized. See root `AGENTS.md` first, then `wiki/_master-index.md` for navigation.

## What lives here

- `_master-index.md` — entry point for all topic hubs
- `<topic>/_index.md` — hub for each topic (architecture, infrastructure, paperclip-operations, vawn-mix-engine, vawn-project, ai-filmmaking, design-engineering, cross-topic, apulu-prompt-generator)
- Topic notes — focused articles linked from the hub

## What `wiki/` is NOT

- **Not raw research.** Raw research lives in `research/<project>/<phase>/`. Wiki articles are *compiled* from raw research after digestion.
- **Not journals.** Dated operational notes live in `journals/<project>/`.
- **Not pipeline output.** Anything auto-generated by the pipeline is gitignored or lives elsewhere.

## Operating rules

- **Route, don't recurse.** When asked about a topic, find the hub (`wiki/<topic>/_index.md`) first; only descend to specific articles when needed.
- **Hubs are folder-scoped indexes.** Don't add frontmatter to reference notes — folders + hubs are the filter (per `VAULT.md` anti-patterns).
- **Cross-link with `[[wikilinks]].`** Use Obsidian-style wikilinks for inter-note references; avoid duplicating content across notes.
- **Update `_master-index.md`** when adding a new topic hub. Update the topic hub when adding a new article inside it.
- **Compile, don't paste.** When new raw material arrives, *digest* it into existing or new wiki articles. Do not paste raw research dumps into `wiki/`.
- **Prefer the most specific, newest, task-relevant file** when sources conflict.

## Adding a new article

1. Identify the right topic hub (or create one only if no existing hub fits).
2. Write the article in the topic folder with a focused, descriptive filename.
3. Add a one-line link from the topic's `_index.md`.
4. If you created a new hub, add a one-line link from `_master-index.md` and from `VAULT.md`'s domain map.

## Don't touch without explicit ask

- `wiki/_master-index.md` structure beyond adding a new line
- `wiki-archive/` (deprecated content kept for history)
- Any hub `_index.md` reorganization that would break inbound wikilinks
